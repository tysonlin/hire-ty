# Typst CV Generation

This document explains how to use [Typst](https://typst.app) to compile the tailored CV for each job application.

---

## What is Typst?

Typst is a modern, open-source typesetting system (similar to LaTeX but significantly simpler). It takes a `.typ` source file and compiles it into a clean, professional PDF. It is free, fast, and produces ATS-compatible output.

---

## Installation

### macOS

```bash
brew install typst
```

> If you do not have Homebrew: visit https://brew.sh and follow the one-line install command shown there, then run the command above.

### Windows

1. Download the latest release from https://github.com/typst/typst/releases
2. Choose the file named `typst-x86_64-pc-windows-msvc.zip` (or `arm64` for ARM devices).
3. Extract the zip and move `typst.exe` to a folder on your `PATH` (e.g. `C:\Windows\System32` or create `C:\tools` and add it to your environment `PATH`).
4. Open a new terminal and verify: `typst --version`

### Linux

```bash
# Debian/Ubuntu
sudo apt install typst

# Arch Linux
sudo pacman -S typst

# Or via the official installer (any distro):
curl -fsSL https://typst.app/install.sh | sh
```

### Verify installation

```bash
typst --version
```

You should see output like `typst 0.11.x`. If you see "command not found", revisit the installation steps.

## CV template

Each job application folder (`/jobs/<Company> - <Role> - <Location>/`) contains a `cv.typ` file generated by the agent. This file uses the **`moderner-cv`** community template from the Typst Universe package registry.

> Template reference: https://typst.app/universe/package/moderner-cv

The agent pre-populates the template with all content resolved by the sub-agent pipeline. You should not need to edit `cv.typ` manually, but you can open it in any text editor if you want to make manual adjustments.

### Template package — first-time setup

Typst downloads packages automatically on first use. When you compile for the first time, it will fetch the `moderner-cv` package from the registry. You need an internet connection for this step.

---

## Compiling the CV

Navigate to the job folder and run:

```bash
cd "jobs/<Company Name> - <Job Title> - <City>, <Country>"
typst compile cv.typ cv.pdf
```

**Example:**

```bash
cd "jobs/Acme Corp - Senior Software Engineer - Sydney, Australia"
typst compile cv.typ cv.pdf
```

This produces `cv.pdf` in the same folder.

### Watch mode (live preview while editing)

If you are iterating on the CV manually:

```bash
typst watch cv.typ cv.pdf
```

Typst will recompile every time you save `cv.typ`. Open `cv.pdf` in a PDF viewer that supports live reload (e.g. Preview on macOS, Evince on Linux, or the Typst VS Code extension).

### VS Code extension (optional, recommended)

Install the **Tinymist Typst** extension in VS Code for syntax highlighting, live preview in the editor, and one-click compile:

1. Open VS Code → Extensions (`Ctrl+Shift+X` / `Cmd+Shift+X`).
2. Search for `Tinymist`.
3. Install **Tinymist Typst** by Myriad-Dreamin.
4. Open any `.typ` file → click the preview icon (top-right) or press `Ctrl+Shift+P` → "Typst Preview".

## CV template structure

The `cv.typ` file generated by the agent follows this section order (matching the `moderner-cv` template):

```
Header (name, contact, links)
Summary
Experience
Education
Skills
Certifications (if applicable)
Projects (if applicable)
```

Do not reorder sections without re-running the ATS Analyzer sub-agent, as ATS systems expect standard section ordering.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `typst: command not found` | Installation did not complete — revisit the Installation section above |
| `error: package not found` | Check your internet connection; Typst needs to download the template on first use |
| PDF looks blank | Open `cv.typ` in a text editor and check for syntax errors (lines starting with `//error`) |
| Characters not rendering | The template uses a system font. On Linux, install `fonts-liberation` or `fonts-noto` if characters appear as boxes |
| PDF not updating in viewer | Close and reopen the PDF; some viewers cache the old version |
